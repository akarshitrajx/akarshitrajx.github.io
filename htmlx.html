📌 1. HTML BASICS

🏠 HTML HOME

Welcome to the complete HTML guide. HTML is the standard markup language for Web pages.

📖 HTML Introduction

HTML = HyperText Markup Language. It describes the structure of a web page.

  • HTML elements tell the browser how to display content
  • HTML consists of a series of elements
  • HTML elements are represented by tags

✏️ HTML Editors

You can write HTML using:

  • VS Code (recommended)
  • Notepad++
  • Sublime Text
  • Atom
  • Online editors: CodePen, JSFiddle

📄 HTML Basic

A simple HTML document:

<!DOCTYPE html>
<html>
<head>
    <title>My First Page</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p>This is my first paragraph.</p>
</body>
</html>

🧩 HTML Elements

An HTML element is everything from the start tag to the end tag:

<tagname>Content goes here...</tagname>
  • <h1>Heading</h1> — heading element
  • <p>Paragraph</p> — paragraph element
  • Empty elements (no closing tag): <br>, <hr>

🏷️ HTML Attributes

Attributes provide additional information about elements:

<a href="https://example.com">Link</a>
<img src="photo.jpg" alt="My Photo" width="300" height="200">
<p class="intro" id="para1">This is a paragraph.</p>
  • href — link destination
  • src — image source
  • alt — alternative text
  • width / height — size
  • class — CSS class
  • id — unique identifier
  • style — inline CSS

📦 HTML CSS

Three ways to add CSS:

  • Inline: <h1 style="color:red;">
  • Internal: <style> inside <head>
  • External: <link rel="stylesheet" href="style.css">
<!-- External CSS -->

<!-- Internal CSS -->
<style>
    body { background: #f0f0f0; }
    h1 { color: navy; }
</style>

📦 Block vs Inline

Block-level: Takes full width, starts new line

<div>, <h1>, <p>, <ul>, <table>, <form>

Inline: Takes only needed width, stays in line

<span>, <a>, <img>, <strong>, <em>, <input>

🧠 HTML Semantics

Semantic elements clearly describe their meaning to browser and developer:

  • <header> — Introduction / top
  • <nav> — Navigation links
  • <main> — Main content
  • <article> — Independent content
  • <section> — Thematic grouping
  • <aside> — Sidebar
  • <footer> — Bottom / credits
  • <figure> — Image with caption
  • <figcaption> — Caption for figure
  • <time> — Date/time
  • <mark> — Highlighted text

  • ✅ Use alt for images

⚖️ HTML vs XHTML

  • HTML: More flexible, case-insensitive, forgiving
  • XHTML: Stricter, XML-based, all tags must close, lowercase
  • XHTML requires <br /> instead of <br>
  • Modern websites use HTML5 (not XHTML)

🌐 HTML Web APIs

JavaScript APIs that work with HTML:

  • Geolocation — user location
  • Drag/Drop — drag elements
  • Web Storage — localStorage, sessionStorage
  • Web Workers — background scripts
  • SSE — Server-Sent Events

📍 HTML Geolocation

<button onclick="getLocation()">Get Location</button>
<p id="demo"></p>

<script>
    function getLocation() {
        if (navigator.geolocation) {
            navigator.geolocation.getCurrentPosition(showPosition);
        }
    }
    function showPosition(position) {
        document.getElementById("demo").innerHTML =
            "Latitude: " + position.coords.latitude +
            " Longitude: " + position.coords.longitude;
    }
</script>

🔄 HTML Drag/Drop

<div id="dropzone" ondragover="allowDrop(event)" ondrop="drop(event)">
    Drop here
</div>
<img id="dragme" src="photo.jpg" draggable="true" ondragstart="drag(event)">

<script>
    function allowDrop(e) { e.preventDefault(); }
    function drag(e) { e.dataTransfer.setData("text", e.target.id); }
    function drop(e) {
        e.preventDefault();
        const data = e.dataTransfer.getData("text");
        e.target.appendChild(document.getElementById(data));
    }
</script>

💾 HTML Web Storage

// localStorage — persists even after browser closes
localStorage.setItem("name", "Akarshit");
localStorage.getItem("name");
localStorage.removeItem("name");

// sessionStorage — persists only for the session
sessionStorage.setItem("theme", "dark");
sessionStorage.getItem("theme");

📨 HTTP Messages

HTTP Status Codes:

  • 1xx: Information (100 Continue)
  • 2xx: Success (200 OK, 201 Created)
  • 3xx: Redirection (301 Moved, 304 Not Modified)
  • 4xx: Client Error (404 Not Found, 403 Forbidden)
  • 5xx: Server Error (500 Internal Server Error, 502 Bad Gateway)

🔧 HTTP Methods

  • GET: Request data from server
  • POST: Send data to server
  • PUT: Update data
  • DELETE: Delete data
  • PATCH: Partial update
  • HEAD: Headers only
  • OPTIONS: Supported methods
