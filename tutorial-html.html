🚀 Introduction

HTML (HyperText Markup Language) web pages ka skeleton hai. Har website HTML se shuru hoti hai. Ye sirf text nahi — structure, meaning, aur content organize karta hai.

Kya seekhenge:

  • HTML document structure (DOCTYPE, html, head, body)
  • Text formatting, headings, paragraphs
  • Links, images, lists, tables
  • Forms — user input handle karna
  • Semantic HTML5 tags (header, nav, section, article, footer)
  • Meta tags, SEO, Accessibility
Tools: Koi bhi text editor chalega — Notepad, VS Code, Sublime Text. Browser mein file open karo aur result dekho!

🏗️ Basic Structure

HTML Document Skeleton

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Page</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <!-- Content yahan aayega -->
    <h1>Hello, World!</h1>
    <p>This is my first web page.</p>
    <script src="script.js"></script>
</body>
</html>

Essential Tags Explained

TagPurpose
<!DOCTYPE html>HTML5 document declare karta hai
<html>Root element, sab kuch iske andar
<head>Meta info, title, styles, scripts (visible nahi)
<body>Visible content yahan
<title>Browser tab ka title
<meta charset>Character encoding (UTF-8 = universal)
<meta viewport>Mobile responsive banana
<link>CSS file attach karna
<script>JavaScript attach karna

📝 Text Formatting

Headings

<h1>Main Heading (Sabse bada)</h1>

Paragraphs & Line Breaks

Text Formatting Tags

🔗 Links & Navigation

Basic Links

<!-- External link -->
<a href="https://google.com">Google par jao</a>

<!-- New tab mein open -->
<a href="https://google.com" target="_blank" rel="noopener noreferrer">
    New Tab mein Google
</a>

<!-- Same page link -->
<a href="about.html">About Us</a>
<a href="contact.html">Contact</a>

<!-- Email link -->
<a href="mailto:akarshit@example.com">Email bhejo</a>

<!-- Phone link (mobile pe call karega) -->
<a href="tel:+919876543210">Call karo</a>

<!-- Download link -->
<a href="file.pdf" download>PDF Download karo</a>

Page Navigation (Anchors)

<!-- Navigation menu -->
<nav>
    <a href="#home">Home</a>
    <a href="#about">About</a>
    <a href="#services">Services</a>
    <a href="#contact">Contact</a>
</nav>

<!-- Sections with IDs -->
<section id="home">
    <h2>Home</h2>
    <p>Home section content...</p>
</section>

<section id="about">
    <h2>About</h2>
    <p>About section content...</p>
    <a href="#home">↑ Back to top</a>
</section>

Link States (CSS ke saath)

/* CSS mein link states style karte hain */
a:link { color: blue; }      /* Unvisited */
a:visited { color: purple; } /* Visited */
a:hover { color: red; }      /* Mouse hover */
a:active { color: green; }   /* Click ke waqt */

🏛️ Semantic HTML5

Semantic tags content ka meaning batate hain. Ye SEO aur accessibility ke liye best hain.

Complete Semantic Page Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Semantic Page</title>
</head>
<body>
    <!-- Header -->
    <header>
        <h1>Website Name</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
                <li><a href="#contact">Contact</a></li>
            </ul>
        </nav>
    </header>

    <!-- Main Content -->
    <main>
        <!-- Article -->
        <article>
            <header>
                <h2>Article Title</h2>
                <time datetime="2026-06-08">June 8, 2026</time>
            </header>
            <p>Article content...</p>
            <section>
                <h3>Section 1</h3>
                <p>Section content...</p>
            </section>
            <section>
                <h3>Section 2</h3>
                <p>More content...</p>
            </section>
            <footer>
                <p>Written by Akarshit Raj</p>
            </footer>
        </article>

        <!-- Sidebar -->
        <aside>
            <h3>Related Articles</h3>
            <ul>
                <li><a href="#">Article 1</a></li>
                <li><a href="#">Article 2</a></li>
            </ul>
        </aside>
    </main>

    <!-- Footer -->
    <footer>
        <p>&copy; 2026 My Website. All rights reserved.</p>
        <address>
            Contact: <a href="mailto:info@example.com">info@example.com</a>
        </address>
    </footer>
</body>
</html>

TagUse
<header>Page/section header (logo, nav, title)
<nav>Navigation links
<main>Main content (page mein sirf ek)
<article>Independent content (blog post, news)
<section>Thematic group of content
<aside>Sidebar, related content
<footer>Page/section footer (copyright, links)
<figure>Image, diagram, chart + caption
<figcaption>Figure ka caption
<details>Collapsible content
<summary>Details ka heading
<mark>Highlighted text
<time>Date/time
<address>Contact information
<progress>Progress bar
<meter>Scalar measurement

♿ Accessibility (A11y)

Accessibility ensure karti hai ki aapki website sab users ke liye accessible ho — including visually impaired, motor disabled, aur keyboard users.

Key Principles

  • Perceivable: Information sab ko mil sake
  • Operable: Sab interface elements use kar sake
  • Understandable: Content clear ho
  • Robust: Assistive technologies ke saath kaam kare

Alt Text for Images

<!-- Good -->
<img src="chart.jpg" alt="Bar chart showing 50% increase in sales from 2024 to 2026">

<!-- Decorative image (empty alt) -->
<img src="decoration.jpg" alt="">

<!-- Complex image with detailed description -->
<figure>
    <img src="diagram.jpg" alt="Flowchart of user registration process">
    <figcaption>
        Detailed description: Step 1: User enters email. 
        Step 2: Verification email sent. Step 3: User clicks link...
    </figcaption>
</figure>

ARIA Labels

<!-- Icon buttons without text -->
<button aria-label="Close menu">
    <span class="icon-close"></span>
</button>

<!-- Current page indicator -->
<nav aria-label="Main navigation">
    <ul>
        <li><a href="/" aria-current="page">Home</a></li>
        <li><a href="/about">About</a></li>
    </ul>
</nav>

<!-- Live regions (dynamic content) -->
<div role="alert" aria-live="assertive">
    Form submitted successfully!
</div>

<!-- Landmark roles -->
<div role="banner">...</div>       <!-- Header -->
<div role="navigation">...</div>   <!-- Nav -->
<div role="main">...</div>        <!-- Main -->
<div role="complementary">...</div> <!-- Aside -->
<div role="contentinfo">...</div>  <!-- Footer -->

Form Accessibility

<!-- Always associate label with input -->
<label for="email">Email Address</label>
<input type="email" id="email" name="email" required 
       aria-required="true" aria-describedby="email-help">
<span id="email-help">We'll never share your email.</span>

<!-- Error message -->
<input type="email" id="email" aria-invalid="true" 
       aria-describedby="email-error">
<span id="email-error" role="alert">Please enter a valid email.</span>

<!-- Fieldset for grouped inputs -->
<fieldset>
    <legend>Choose your preferred contact method</legend>
    <input type="radio" id="email-contact" name="contact">
    <label for="email-contact">Email</label>
    <input type="radio" id="phone-contact" name="contact">
    <label for="phone-contact">Phone</label>
</fieldset>

Keyboard Navigation
